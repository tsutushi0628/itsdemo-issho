Editor Spotlighter - スマホからClaude Codeを操作するリモートチャット機能の設計書

概要: PCで起動中のClaude Code CLIを、スマホのブラウザからチャット形式で操作する。 ファイル編集はPC上のCLIが自律的に行い、スマホでは「会話」だけに集中する。

キーワード: WebSocket中継 / CLIのstdin・stdout転送 / xterm.jsレンダリング / トークン認証 / LAN内通信

通信の流れ

  • Editor SpotlighterのVSCode拡張が child_process.spawn('claude', ['--continue']) でCLIを起動
  • CLIの stdout → WebSocket → スマホに表示
  • スマホの入力 → WebSocket → CLIの stdin に流し込み
  • スマホ側は xterm.js でターミナル出力をそのまま綺麗にレンダリング（Claude Code拡張風のガワ）
  • 接続は Tailscale または同一LAN内で行う
  • 認証は トークン認証（ランダム生成）で保護

4. PCに戻る体験（/spContinue）

スマホで進めた会話をPCのVSCode拡張に引き継ぐためのコマンド。

📱
Step 1: スマホで作業を進める 移動中・外出先でClaude Codeにチャットで指示を出す
💻
Step 2: PCに戻ってVSCodeを開く コマンドパレットで /spContinue を実行
🔄
Step 3: 会話の引き継ぎ スマホセッションのCLIプロセスをVSCode拡張側のターミナルに接続し直す。
スマホ側には「PCに接続が移りました」と表示し、WebSocketを切断。
✅
Step 4: PCで続行 Editor Spotlighterの通常UIでそのままClaude Codeの会話を継続。
--continue フラグにより、会話履歴は途切れない。

6. セキュリティ

🔐 トークン認証

crypto.randomBytes(32) でサーバー起動時にトークンを生成。 URLクエリパラメータで渡し、WebSocket接続時に検証する。トークンは毎回変わる。

🌐 ネットワーク制限

デフォルトはLAN内（192.168.x.x / 10.x.x.x）のみ接続許可。 Tailscale経由なら外出先からも安全にアクセス可能。

⚙ allowExternalAccess 設定

VSCode設定で editorSpotlighter.allowExternalAccess を有効にした場合のみ、 LAN外からの接続を許可する。デフォルトは false。

📷 QRコード接続

接続URLを手入力する必要がないよう、VSCode上にQRコードを表示。 スマホのカメラで読み取るだけで接続できる。トークンもQRに含まれる。
