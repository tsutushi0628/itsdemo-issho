# Implementation Log: タスク1〜19実装（2026-06-11）

## 概要

カラム単位リモート操作（column-remote-control）のタスク1〜19を経路C（実装2名並列＋QA＋メインレビュー）で実装した。タスク20・21（実機回帰・実機E2E）は未実施で残置。

## 実装サマリ

- フォーカスルーティング: `src/remote/focusRouter.ts`（新規）。FOCUS_GROUP_COMMANDS 8列・Claude タブ判定・50ms間隔ポーリング検証（[4] 1500ms / [5] 500ms）・fail-closed。
- 注入パイプライン: `src/remote/injectionPipeline.ts`（新規）。design 2.3 [0]-[10] 直列・クリップボード退避/復元・bundle identifier 完全一致の単一 osascript（モジュール定数）・タイムアウト時の子プロセス kill。
- サーバ状態API: `src/remote/remoteViewServer.ts`。clampSelectedColumn / deriveColumnLabels / buildColumnsPayload / setColumns（同一ペイロードスキップ）/ sendColumnsTo / sendInjectResult。selectColumn は全分岐で columns 応答（ACK契約）。frame に column（クロップ元列番号）を付与。
- モバイルUI: `src/remote/mobileHtml.ts`。列バー動的生成（textContent 固定）・送信先表示・切替中オーバーレイ（pendingColumn 相関ACK＋フレーム列ゲート）・注入結果トースト（固定文言マップ・hasOwnProperty ガード）・maxlength=2000・失敗時本文復元・再接続時状態リセット。
- 待ち受け安全化: bind 既定 127.0.0.1（DEFAULT_BIND_ADDRESS を qrPolicy から一元 export）・qrPolicy 純関数（具体IP bind はそのIPでURL生成）・サイドバー localOnly 表示・初回移行案内（config.inspect で明示設定者には非表示・fire-and-forget でアクティベーション非ブロック）。

## 実機確定事項

- **Claude Code タブの viewType 実値**（タスク5）: ワークスペースストレージDB（`state.vscdb` の `memento/workbench.parts.editor`）の実測で `"viewType":"mainThreadWebview-claudeVSCodePanel"`・`"providedId":"claudeVSCodePanel"` を15ワークスペースで一貫確認。判定定数 `claudeVSCodePanel` の包含判定で整合。
- **AppleScript 属性名**: `bundleID` は実在せず実行時エラー（-1700）になることを実機 osascript で確認。`bundle identifier` が正。修正後の比較スクリプト構造（keystroke 除去版）は前面 Chrome 状態で `ABORT:com.google.Chrome` を返し完全一致ガードの動作を確認。
