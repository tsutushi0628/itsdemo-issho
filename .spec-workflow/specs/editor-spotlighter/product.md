# Product: Editor Spotlighter

## Product Purpose

VSCodeのエディタカラム幅を自動制御するVSCode拡張。フォーカスしたカラムが広がり他が縮む「アコーディオン動作」と、画面サイズに応じたアクティブカラム数の自動調整を行う。マルチモニター環境（ウルトラワイド〜ノートPC）で最適なレイアウトを提供。

## Target Users

マルチモニター環境で開発するプログラマー。特にウルトラワイドモニターとノートPCを併用する開発者。

## Key Features

1. **レイアウト整形ボタン** - エディタアクション（エディタ右上）にボタン配置。押すと設定どおりにカラム幅が等間隔に揃う。その後はユーザーが自由にドラッグで調整可能。もう一度押すと再整形
2. **アコーディオンモード** - 画面が狭いとき（activeColumns < totalColumns）のみ自動発動。フォーカスしたカラムが広がり他が縮む。ウルトラワイドでは発動しない
3. **タブサイドバーパネル** - アクティビティバーにESアイコン。クリックでサイドバーにカラムごとにグループ化されたタブ一覧を表示。タブクリックでそのエディタにフォーカス。パネル上部にレイアウト整形ボタンも配置
4. **タブ設定UI** - 設定画面のチェックボックスでタブの開き方を制御（アクティブタブの右隣に開く、プレビューモード無効化）
5. **モニター解像度検知** - macOS system_profilerでメインディスプレイの解像度を取得し、プリセットに基づいてアクティブカラム数を自動決定

## Product Principles

1. **設定より規約** - デフォルト値だけで使い始められる（totalColumns=4、activeRatio=0.35、inactiveRatio=0.1）
2. **壊れたら即座にわかる** - 解像度検出失敗時はWarningMessageで通知、エラーは握りつぶさず再throw
3. **既存のVSCode体験を壊さない** - deactivate時にレイアウトを等幅にリセット、toggle/resetLayoutコマンドでいつでも元に戻せる
4. **ユーザーの自由を奪わない** - 自動整形はデフォルトoff、手動ボタンがメイン
