# Product: Editor Spotlighter

## Product Purpose

VS Code のエディタカラムレイアウトを自動制御し、加えてスマホからの遠隔閲覧・操作を提供する VS Code 拡張。フォーカスしたカラムが広がり他が縮む「アコーディオン動作」と、ウィンドウ幅に応じたアクティブカラム数の自動調整を中核に、レイアウトが崩れても読み戻し照合で自己回復する。さらに、外出先のスマホから PC の VS Code 画面をリアルタイム閲覧し、特定カラムの Claude Code セッションへ入力をルーティングできる。ウルトラワイドからノート PC まで、画面サイズごとに手作業でレイアウトを調整する手間を解消することが目的。

## Target Users

マルチモニター環境で開発するプログラマー。特にウルトラワイドモニター・ノート PC・外部モニターを使い分け、画面サイズが変わるたびにカラムレイアウトを手で調整していた開発者。加えて、移動中や離席中もスマホから自分の Claude Code セッションを確認・操作したい開発者。現状はローカル個人運用が前提。

## Key Features

1. **レイアウト整形ボタン** - エディタ右上のアクションとタブ一覧パネル上部にボタンを配置。押すと全カラムが設定どおり等間隔に揃い、フォーカス履歴もリセットする。その後はユーザーが自由にドラッグで調整でき、もう一度押すと再整形できる。

2. **アコーディオンモード** - 画面が狭いとき（アクティブ列数が総列数未満のとき）に自動発動。直近にフォーカスした列がアクティブ列数の上限まで広がり、他は固定幅に縮む（直近にフォーカスした列を先頭に保持し、上限を超えると最も古い列から押し出す方式）。アクティブ列数が総列数に達する広い画面では履歴に依らず全列を等間隔で表示し、この等間隔状態でも崩れを検知して整形を適用する。

3. **レイアウトの読み戻し自己回復** - レイアウト適用後に実際のグリッド状態を読み戻し、要求比率と許容差内で一致しなければ段階的に回復する（全列均等化 → サイドバー再描画 → 幅再実測）。ディスプレイ切替後に内部グリッドが旧幅で固着する問題に対し、「適用成功 ≠ 反映」を前提に閉ループで整える。

4. **タブ一覧サイドバーパネル** - アクティビティバーに専用アイコンを配置。クリックでサイドバーにカラム（タブグループ）ごとに分類したタブ一覧をツリー表示する。タブをクリックするとそのエディタにフォーカスし、各タブから閉じる操作もできる。パネル上部にはレイアウト整形ボタンも置く。

5. **タブの開き方設定** - 設定画面のチェックボックスで「新しいタブをアクティブタブの右隣に開く」「プレビューモードを無効化して常に新タブで開く」を制御。実体は VS Code 本体設定（タブの表示位置・プレビュー有効化）への薄い連動で、二重管理を避ける。推奨設定をまとめて適用するコマンドも備える。

6. **ウィンドウ幅検知によるアクティブ列数の自動決定** - macOS では CGWindowList API（Swift 経由・アクセシビリティ権限不要）、Windows では PowerShell、Linux では xdotool で VS Code ウィンドウの実幅を取得する。実幅からサイドバー幅と画面装飾分を差し引いた編集領域幅を求め、「最小許容幅で底支え・同時アクティブ上限で頭抑え・全幅しきい値以上で全列等間隔」の 3 条件でアクティブ列数を決める。実測は OS プロセス呼び出しで時間がかかるため、ホットパスではキャッシュ値で即時計算し、最短間隔を空けて裏で再実測する。

7. **リモートビュー（スマホ閲覧）** - スマホから PC の VS Code 画面をリアルタイム閲覧する。PC のレイアウトは一切変えず、ウィンドウをキャプチャしてパネル境界を自動検出し、選択中のカラムだけを切り出してスマホ幅にリサイズして配信する。接続中のみ約 1.5 秒間隔で定期キャプチャし、生フレームと切り出し後フレームの二段の変化検知で、画面が変わったときだけ送信する（無変化時は送らず帯域・処理を節約）。

8. **リモートビュー（カラム単位の入力ルーティング・誤爆ゼロ fail-closed）** - スマホで選んだカラムの Claude Code セッションへテキスト入力をルーティングする。注入は順序保証パイプラインで、多重実行の抑止 → 対象列の有効性確認 → 対象列へのフォーカス移動と確定検証 → 直前再検証 → クリップボード退避 → 前面アプリが VS Code 系か（バンドル ID 完全一致で）確認 → 貼り付け＋実行 → クリップボード復元、の各段を通す。OS 全体に効く入力操作のため、一段でも条件を満たさなければ何もせず中止する fail-closed 設計で、別アプリへの誤入力を防ぐ。閲覧専用にしたい場合は設定でリモート入力を無効化できる。

9. **リモートビュー（QR スキャン自動ログイン）** - 起動ごとに高エントロピーのワンタイムパスワードを自動生成し、QR コードに埋め込む。スマホでスキャンするとパスワード入力なしでログインできる。認証キーは QR の URL フラグメントにのみ載せ、HTTP リクエストやサーバ・トンネルのログ、ブラウザ履歴に残さない（読み込み直後にアドレスバーから除去）。サイドバーに表示する URL には鍵を含めない。

10. **リモートビュー（待ち受けと認証）** - 待ち受けアドレスの既定はローカルのみ（127.0.0.1）で、Cloudflare Tunnel 経由の HTTPS アクセスを推奨する。同一 LAN への直結は明示設定で全インターフェース待ち受けに切り替えるが、平文 HTTP のため盗聴リスクがある旨を案内する。アクセス許可は偽装可能なヘッダを使わず実ソケット IP のみで判定し、ループバックとプライベート IP 帯だけを通す。認証はパスワード＋Cookie セッション（一定時間で失効）で、ログイン連続失敗時は同一 IP を一時ロックアウトする。

## Business Objectives

開発者体験の改善。画面サイズごとのレイアウト調整と、外出先からのセッション確認・操作にかかる手間を削減する。現状はオーナー個人のローカル運用が前提で、ビルド時に拡張フォルダへ自動配置して使う。

## Success Metrics

- レイアウト崩れに対する手動整形操作の削減（自己回復で再整形の必要を減らす）。
- フォーカスからレイアウト反映までの体感遅延の短縮。
- スマホからのセッション閲覧・入力が誤爆なく成立すること。

## Product Principles

1. **ユーザーの自由を奪わない** - 自動整形が前提を壊さないこと。手動の整形ボタンをメイン操作とし、その後のドラッグ調整を妨げない。アクティブ列数の自動決定は最小幅で底支えし、上限で頭抑えする。

2. **設定より規約** - デフォルト値だけで使い始められる（総列数 5・最小許容幅 460px・全幅しきい値 3000px・同時アクティブ上限 2・サイドバー差し引き幅 230px・待ち受けはローカルのみ）。

3. **適用は読み戻し照合の閉ループで組む** - 「適用が成功した」ことと「実際に反映された」ことを区別する。レイアウトは読み戻して照合し、不一致なら署名を残さず次のフォーカスで必ず再適用する。

4. **重要操作は失敗を表面化し、補助操作は静かに巻き戻す** - レイアウト適用・推奨設定・タブ設定の失敗は throw して気づけるようにする。サイドバー開閉・キャプチャ・QR 生成などの補助処理は失敗してもリトライ可能な状態に巻き戻して継続する。

5. **遠隔操作は fail-closed で誤爆を防ぐ** - 入力注入は OS 全体に効くため、列フォーカスの確定検証と前面アプリのバンドル ID 完全一致を通った場合だけ実行し、一段でも崩れたら何もしない。

6. **既存の VS Code 体験を壊さない** - 無効化時・終了時にレイアウトを等間隔へ戻し、トグル／整形コマンドでいつでも元に戻せる。タブ設定は本体設定の薄いラッパーとして二重管理を避ける。

7. **秘密を残さない** - リモートアクセスの認証キーは URL フラグメントにのみ載せ、リクエスト・ログ・履歴に残さない。パスワードは起動ごとのワンタイム生成を推奨し、固定値は撮影・画面共有のリスクとして案内する。

## Future Vision

- リモートビューの全機能（キャプチャ・入力注入・貼り付け）が現状 macOS 専用のため、他プラットフォームへの対応拡大。
- レイアウトプリセットの保存・切替。
- キーボードショートカット。
- 個人ローカル運用から、配布・公開を見据えた運用形態の検討。

## Known Limitations

- リモートビューのキャプチャ・入力注入・貼り付けは macOS 専用（ウィンドウ ID 取得・画面キャプチャ・前面アプリ操作が macOS 固定）。ウィンドウ幅検知のみ 3 プラットフォーム対応。
- 同一 LAN への直結は平文 HTTP（TLS 未対応）で盗聴され得るため、トンネル経由 HTTPS を推奨。
- キャプチャ画像とデバッグログを一時ディレクトリの固定パスへ書き出す（デバッグログは上限超過時に後半のみ残すローテーションあり）。
- 配布物の除外設定でソース・各種設定は除外されるが、調査メモ等のドキュメント類とソースマップは現状除外対象に含まれていない。
